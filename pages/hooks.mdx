# Hooks

This page explains the usage of all react hooks exported by `@kanji-world/react-sdk` package.

## Hook Types
Hooks in the SDK can be categorized in two types:
1. Custom ones that communicate with Kanji API;
2. `@wagmi/cli` generated ones.

`@wagmi/cli` is a tool created by [`wagmi`](https://wagmi.sh/) that reduces the boilerplate of writing SOLID compliable React Hooks.

Our configuration generates `PrepareWrite`, `Write`, and `Read` hooks.

Hook names are built with contract names and function names that you wan't to trigger, wether it's a read or a write function.

The documentation covers only the custom hooks that actually implement an undocumented logic, because `@wagmi/cli` generated hooks pretty much derive the logic from the parent hooks they use underhood:
- [`useContractWrite`](https://wagmi.sh/react/hooks/useContractWrite)
- [`useContractRead`](https://wagmi.sh/react/hooks/useContractRead)
- [`usePrepareContractWrite`](https://wagmi.sh/react/prepare-hooks/usePrepareContractWrite)

Considering that generated hook list is pretty long (around 396+ hooks), there is no meaningful reason to repeat the documentation over and over, so please examine and address to `wagmi` doc if you are questioning on how to call a hook.
